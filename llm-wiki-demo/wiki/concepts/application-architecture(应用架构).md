# 应用架构

> English: Application Architecture；状态：`single-source`。

## 定义

应用架构描述支持业务和数据需要的应用服务、逻辑应用组件、物理应用组件、功能分布及相互关系。它把应用组合连接到业务服务、能力、流程、数据、用户、地点和运营组织。

## 在 ADM 中的作用

阶段 C 盘点并合理化应用组合：拆分过于复杂的应用、消除重复功能、区分逻辑需求和物理产品，并形成基线和目标应用架构。其差距和依赖产生路线图组件，并向技术架构提出平台、部署、集成和运维约束。

应用与数据架构可以任一顺序启动，也可在发现数据责任、集成或模型冲突时短周期往返。

## 边界

应用架构不是单个应用的详细软件设计。它关注组合级结构、职责和关系，详细模块或配置只在支持架构决策时出现。
