# BM25 工作流

BM25 是面向较大型 LLM Wiki 的可选本地全文搜索层。它是导航层，不是事实来源。

## 加载偏好

在 bootstrap、ingest、query、lint 或 BM25 操作前，按照 `references/config/extend-schema.md` 中的优先级顺序读取 `EXTEND.md`。

如果不存在 `EXTEND.md`，执行首次偏好设置，并根据 `references/templates/extend.md` 创建该文件。不要静默使用默认值。

## 触发检查

当 `bm25.mode: auto_prompt` 时，计算：

- `raw/` 下的文件数，不包括 `.gitkeep`
- `wiki/` 下的 Markdown 文件数
- `wiki/**/*.md` 的总字符数
- `wiki/index.md` 的行数
- 最近一次查询读取的页面数（如果已知）

如果达到 `bm25.thresholds` 中的任一常规阈值，询问是否初始化 BM25。如果 `prompt_once: true`，并且用户已在常规阈值下拒绝，则在达到强阈值前不要再次询问。

如果达到任一强阈值，即使用户之前拒绝过，也要再次询问。

`bm25.mode` 的处理方式：

- `manual`：不要主动询问；仅在用户明确要求时初始化。
- `off`：不要检查或询问。
- `enabled`：如果 BM25 已存在则使用；如果缺失则询问是否初始化。
- `required`：ingest、query 和 lint 必须使用 BM25；如果不可用则停止。

## 初始化

仅在用户明确批准后初始化。

创建或更新：

```text
scripts/wiki_fts.py
indexes/
indexes/README.md
exports/
.gitignore
SCHEMA.md
wiki/log.md
```

使用以下模板：

- `references/templates/wiki_fts.py` -> `scripts/wiki_fts.py`
- `references/templates/bm25-readme.md` -> `indexes/README.md`

`references/templates/wiki_fts.py` 位于 templates 下，因为它会被复制到生成的 wiki 中。它不是从已安装 skill 包中直接执行的 skill 自有辅助脚本。

如果缺少以下忽略规则，则追加：

```gitignore
indexes/fts.sqlite
exports/*.jsonl
exports/*.csv
exports/*.md
```

然后运行：

```bash
python3 scripts/wiki_fts.py doctor
python3 scripts/wiki_fts.py build
python3 scripts/wiki_fts.py stats
python3 scripts/wiki_fts.py search "LLM Wiki" --limit 5
python3 scripts/wiki_fts.py export --format jsonl --out exports/bm25-chunks.jsonl
```

如果冒烟测试的查询词没有出现在 wiki 中，从 `wiki/index.md` 选择一个标题或概念。

## 更新 Schema

向 `SCHEMA.md` 添加 `Search Layer` 小节（保留该结构性标题）：

````markdown
## Search Layer

BM25 是可选的本地组件，帮助 LLM 查找候选 wiki 段落。
它不是事实来源，也不能替代阅读 wiki 页面。

启用 BM25 时：
- Query：读取 `wiki/index.md`，运行 `python3 scripts/wiki_fts.py stats`；如果 `Fresh: no` 则重建；运行 `python3 scripts/wiki_fts.py search "{query}" --limit 10`；打开返回的页面；跟随相关 wikilink；然后使用 wiki 页面引用作答。
- Ingest：创建新的实体、概念、主张、方法或综合页面前，使用 BM25 搜索现有相关页面，以避免重复。
- Lint：运行 `python3 scripts/wiki_fts.py stats`；如果索引过期，重建或记录警告。
- 回退：如果 BM25 失败且偏好允许回退，使用 `rg` 和 `wiki/index.md`。

Normal Query 指 LLM 问答：使用 `stats` 检查新鲜度，尽可能重建过期索引，使用 `search`，打开返回的 `page_path` 文件，阅读完整 wiki 上下文，并引用 wiki 页面。Export Use 指运维、审计、迁移、调试或外部分析；当 `search` 可用时，不要把导出文件用作正常查询路径。

搜索输出字段：

| 字段 | 含义 | 用法 |
| --- | --- | --- |
| `page_path` | 包含匹配内容的 wiki 页面。 | 回答前打开此页面。 |
| `heading_path` | 页面内的标题路径。 | 定位相关小节。 |
| `chunk_id` | `{page_path}#{ordinal}` 诊断标识符。 | 不要引用。 |
| `score` | 当前查询的 SQLite FTS5 BM25 分数；越低越相关，值可能为负。 | 仅用于对单次查询的候选结果排序。 |
| `snippet` | 截断的文本块预览。 | 绝不能仅根据该片段回答。 |

绝不要在面向用户的答案中引用 `chunk_id`、`score`、`indexes/fts.sqlite` 或 `exports/*` 作为证据。BM25 数据用于查找候选页面；wiki 页面才是引用目标。

BM25 数据模型记录在 `indexes/README.md` 中。导出行严格包含以下字段：

```text
chunk_id, page_path, title, type, heading_path, ordinal, sources, tags, updated, text
```
````

## Query 中的使用方式

1. 阅读 `wiki/index.md` 了解结构。
2. 提取查询词、实体和可能的页面名称。
3. 运行 `python3 scripts/wiki_fts.py stats`。
4. 如果 `Fresh: no`，运行 `python3 scripts/wiki_fts.py rebuild`；如果重建失败且允许回退，使用 `rg` 和索引导航。
5. 针对用户问题和关键词运行 BM25 搜索。
6. 打开返回的 wiki 页面；不要仅根据摘要片段回答。
7. 跟随相关 wikilink。
8. 使用 wiki 页面引用作答。
9. 如果 BM25 没有有用结果，回退到 `rg` 和索引导航。

## Normal Query 与 Export Use

**Normal Query** 指 LLM 问答。使用 `stats` 检查新鲜度，尽可能重建过期索引，使用 `search`，打开返回的 `page_path` 文件，阅读完整 wiki 上下文，然后引用 wiki 页面。

**Export Use** 指运维、审计、迁移、调试或外部分析。当 `search` 可用时，导出文件不是正常查询路径。

不要在面向用户的答案中引用 BM25 内部信息：不得将 `chunk_id`、`score`、`fts.sqlite` 或 `exports/*` 作为证据。BM25 数据用于定位候选 wiki 页面；wiki 页面才是引用目标。

## 搜索输出契约

`search` 输出经过排序的候选文本块：

```text
1. {title} [{page_path}]
   heading: {heading_path}
   chunk: {chunk_id} score: {score}
   {snippet}
```

| 字段 | 含义 | LLM 用法 |
| --- | --- | --- |
| `title` | 来自 frontmatter 或第一个标题的页面标题。 | 仅用作标签。 |
| `page_path` | 相对于生成的 wiki／项目根目录、包含 `wiki/` 前缀的路径。 | 回答前打开此页面。 |
| `heading_path` | 页面内的标题路径。 | 打开页面后用于定位相关小节。 |
| `chunk_id` | `{page_path}#{ordinal}`。 | 仅作诊断标识；不要引用。 |
| `score` | SQLite FTS5 BM25 分数。越低越相关；值可能为负。 | 仅用于对单次查询的候选结果排序。 |
| `snippet` | 截断的文本块内容。 | 仅用于预览；绝不能仅根据它回答。 |

当前搜索会将查询拆分为词项，并用 OR 组合。为了提高精度，使用 `wiki/index.md` 中的准确页面标题、实体名称或有辨识度的短语执行多次聚焦搜索。

## 数据模型契约

生成的 SQLite 数据库是 `indexes/fts.sqlite`，由 `scripts/wiki_fts.py` 根据 `wiki/` 下的 Markdown 页面构建。当前 schema 版本为 `1`。

### `pages`

`wiki/` 下的每个 Markdown 页面对应一行。

| 字段 | 类型 | 定义 |
| --- | --- | --- |
| `page_path` | `TEXT PRIMARY KEY` | 相对于生成的 wiki／项目根目录、包含 `wiki/` 前缀的路径，例如 `wiki/concepts/rag.md`。 |
| `title` | `TEXT NOT NULL` | Frontmatter 中的 `title`、第一个 `#` 标题，或作为回退的文件名。 |
| `type` | `TEXT NOT NULL` | Frontmatter 中的 `type`。 |
| `updated` | `TEXT NOT NULL` | Frontmatter 中的 `updated`，没有则为空字符串。 |
| `sources` | `TEXT NOT NULL` | 以文本形式保留的原始 frontmatter `sources`。 |
| `tags` | `TEXT NOT NULL` | 以文本形式保留的原始 frontmatter `tags`。 |
| `mtime` | `REAL NOT NULL` | 新鲜度检查所用的文件系统修改时间。 |

### `chunks`

每个被索引的文本块对应一行。

| 字段 | 类型 | 定义 |
| --- | --- | --- |
| `chunk_id` | `TEXT PRIMARY KEY` | 稳定标识符 `{page_path}#{ordinal}`。 |
| `page_path` | `TEXT NOT NULL` | 父页面路径；与 `pages.page_path` 匹配。 |
| `title` | `TEXT NOT NULL` | 复制到文本块的父页面标题。 |
| `type` | `TEXT NOT NULL` | 复制到文本块的父页面类型。 |
| `heading_path` | `TEXT NOT NULL` | 文本块所在小节的标题路径。 |
| `ordinal` | `INTEGER NOT NULL` | 文本块在父页面中的序号，从 1 开始。 |
| `sources` | `TEXT NOT NULL` | 复制到文本块的父页面 `sources`。 |
| `tags` | `TEXT NOT NULL` | 复制到文本块的父页面 `tags`。 |
| `updated` | `TEXT NOT NULL` | 复制到文本块的父页面 `updated`。 |
| `text` | `TEXT NOT NULL` | 用于搜索和导出的、移除 Markdown 标记后的文本块内容。 |

### `chunks_fts`

用于对文本块排序的 SQLite FTS5 虚拟表。

| 字段 | 是否索引 | 定义 |
| --- | --- | --- |
| `chunk_id` | 否（`UNINDEXED`） | 从 `chunks.chunk_id` 复制的诊断连接键。 |
| `title` | 是 | 可搜索的页面标题文本。 |
| `page_path` | 否（`UNINDEXED`） | 为导航返回的页面路径。 |
| `heading_path` | 是 | 可搜索的标题路径文本。 |
| `text` | 是 | 可搜索的主要文本块内容。 |

搜索通过 `chunk_id` 将 `chunks_fts` 连接到 `chunks`。BM25 `score` 在查询时从 `chunks_fts` 计算；不会存储，也不会导出。

### `metadata`

| 键 | 含义 |
| --- | --- |
| `built_at` | 构建索引时的 UTC 时间戳。 |
| `schema_version` | 数据模型版本；当前为 `1`。 |
| `root` | 构建时使用的 wiki 根目录绝对路径。 |
| `page_count` | 已索引的 wiki 页面数。 |
| `chunk_count` | 已索引的文本块数。 |

### 导出字段

导出内容包含 `chunks` 中的行，字段严格按以下顺序排列：

```text
chunk_id, page_path, title, type, heading_path, ordinal, sources, tags, updated, text
```

导出内容不包含 `score`、`mtime` 或 FTS 内部信息。分数仅针对当前查询，并且只由 `search` 计算。

## Ingest 中的使用方式

1. 提取候选实体、概念、方法、数据集和主张。
2. 创建新页面前，使用 BM25 搜索现有页面。
3. 如果现有页面表达的是同一个持久概念，优先更新它。
4. 仅在没有合适页面时创建新页面。
5. 修改 `wiki/` 后，如果 `auto_rebuild_after_ingest: true`，重建索引。

## 导出

使用：

```bash
python3 scripts/wiki_fts.py export --format jsonl --out exports/bm25-chunks.jsonl
python3 scripts/wiki_fts.py export --format csv --out exports/bm25-chunks.csv
python3 scripts/wiki_fts.py export --format markdown --out exports/bm25-report.md
```

导出的文件包含带页面元数据字段和文本块内容的文本块行，不包含静态 BM25 分数。分数按查询计算。
