# BM25 Workflow

BM25 is an optional local full-text search layer for larger LLM Wikis. It is a
navigation layer, not a source of truth.

## Preference Loading

Before bootstrap, ingest, query, lint, or BM25 work, read `EXTEND.md` using the
priority order in `references/config/extend-schema.md`.

If no `EXTEND.md` exists, run first-time preference setup and create one from
`references/templates/extend.md`. Do not silently use defaults.

## Trigger Check

When `bm25.mode: auto_prompt`, compute:

- number of files under `raw/` excluding `.gitkeep`
- number of Markdown files under `wiki/`
- total characters across `wiki/**/*.md`
- line count of `wiki/index.md`
- pages read in the most recent query, when known

If any normal threshold in `bm25.thresholds` is reached, ask whether to
initialize BM25. If `prompt_once: true` and the user already declined at normal
thresholds, do not repeat until a strong threshold is reached.

If any strong threshold is reached, ask again even after a previous decline.

If `bm25.mode` is:

- `manual`: do not prompt; initialize only on explicit request.
- `off`: do not check or prompt.
- `enabled`: use BM25 if present; ask to initialize if missing.
- `required`: require BM25 for ingest/query/lint and stop if unavailable.

## Initialization

Only initialize after explicit user approval.

Create or update:

```text
scripts/wiki_fts.py
indexes/
indexes/README.md
exports/
.gitignore
SCHEMA.md
wiki/log.md
```

Use templates:

- `references/templates/wiki_fts.py` -> `scripts/wiki_fts.py`
- `references/templates/bm25-readme.md` -> `indexes/README.md`

Append these ignore rules if missing:

```gitignore
indexes/fts.sqlite
exports/*.jsonl
exports/*.csv
exports/*.md
```

Then run:

```bash
python scripts/wiki_fts.py doctor
python scripts/wiki_fts.py build
python scripts/wiki_fts.py stats
python scripts/wiki_fts.py search "LLM Wiki" --limit 5
python scripts/wiki_fts.py export --format jsonl --out exports/bm25-chunks.jsonl
```

If the smoke-test query term is not present in the wiki, choose a title or
concept from `wiki/index.md`.

## Schema Update

Add a `Search Layer` section to `SCHEMA.md`:

```markdown
## Search Layer

BM25 is optional and local. It helps the LLM find candidate wiki passages.
It is not a source of truth and does not replace reading wiki pages.

When BM25 is enabled:
- Query: read `wiki/index.md`, run `python scripts/wiki_fts.py search "{query}" --limit 10`, open returned pages, follow relevant wikilinks, then answer with citations to wiki pages.
- Ingest: before creating new entity, concept, claim, method, or synthesis pages, search BM25 for existing related pages to avoid duplicates.
- Lint: run `python scripts/wiki_fts.py stats`; if stale, rebuild or record a warning.
- Fallback: if BM25 fails and preferences allow fallback, use `rg` and `wiki/index.md`.
```

## Log Update

Append:

```text
## [YYYY-MM-DD] bm25-init | Search Layer Enabled
- Preferences: {path to EXTEND.md}
- Database: indexes/fts.sqlite
- Script: scripts/wiki_fts.py
- Pages indexed: {count}
- Chunks indexed: {count}
- Export: exports/bm25-chunks.jsonl
```

## Query Use

1. Read `wiki/index.md` for structure.
2. Extract query terms, entities, and likely page names.
3. Run BM25 search for the user question and key terms.
4. Open the returned wiki pages; do not answer from snippets alone.
5. Follow relevant wikilinks.
6. Answer with citations to wiki pages.
7. If BM25 has no useful result, fallback to `rg` and index navigation.

## Ingest Use

1. Extract candidate entities, concepts, methods, datasets, and claims.
2. Search BM25 for existing pages before creating new pages.
3. Prefer updating an existing page when it represents the same durable idea.
4. Create a new page only when no suitable page exists.
5. After modifying `wiki/`, rebuild if `auto_rebuild_after_ingest: true`.

## Export

Use:

```bash
python scripts/wiki_fts.py export --format jsonl --out exports/bm25-chunks.jsonl
python scripts/wiki_fts.py export --format csv --out exports/bm25-chunks.csv
python scripts/wiki_fts.py export --format markdown --out exports/bm25-report.md
```

Exported files contain the indexed corpus, not static BM25 scores. Scores are
computed per query.
